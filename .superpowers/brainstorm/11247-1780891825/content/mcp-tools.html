MCP tool surface — initial set

Tools the agent can call. NudgeAI hosts an MCP server on a per-session unix socket; the embedded pty inherits NUDGEAI_MCP_SOCKET so claude-code / codex / aider auto-discover it. NudgeAI ships preset MCP config snippets per supported agent.

Plan-phase

render_plan

render_plan(markdown: string, assets?: Asset[]) → {plan_id, url}

Agent passes plan as markdown with embedded mermaid blocks and asset references. NudgeAI templates to styled HTML, renders mermaid → SVG, inlines assets, posts to Plan sub-tab and makes it active. Returns a stable URL for citation.

capture_region

capture_region(hint?: string) → {png_path, width, height}

Triggers the existing region-overlay flow inside the active session. The hint shows as a small caption ("capture the current empty state") so the user knows what to grab. Returns a path the agent can reference in a subsequent render_plan call.

await_plan_decision

await_plan_decision(plan_id) → {decision: "approved"|"commented", comments?: string}

Blocks until the user clicks Approve or Comment on the plan toolbar. If commented, returns the free-text the user typed. Agent loops: revise plan, render_plan again, await_plan_decision again.

Verify-phase

open_preview

open_preview(url: string, select_mode?: bool) → {ready: true}

Loads URL in the Preview sub-tab WKWebView and makes it active. If select_mode is on, the SELECT toggle is engaged and the user can click elements immediately.

await_element_feedback

await_element_feedback() → {text, screenshot_path, selector, outer_html, bbox}

Blocks until the user clicks an element and submits the floating composer. Returns: typed feedback, cropped PNG of the element, computed CSS selector, outerHTML snippet (truncated), bounding box. Agent maps to source and edits.

reload_preview

reload_preview() → {ready: true}

Reloads the current preview URL. Agent calls after it finishes a code change so the user sees the new state.

Meta

get_session_info

get_session_info() → {cwd, session_name, agent, history_dir}

Lets the agent know which session it's in and where to write transcript artifacts. Called once at agent boot.

post_chat

post_chat(message: string) → ok

Optional: lets the agent push a status/explanation line into the right-pane terminal as a styled banner (separate from its own stdout). Useful for "I'm now starting to implement…" hand-offs.

Eight tools total. Pure additive — nothing existing breaks. The agent decides flow; NudgeAI just renders and round-trips user feedback.
