Plan + Verify — end-to-end round-trips

Two near-symmetric loops. User starts the session, agent decides when to render & when to gate on the human.

Plan phase
User
Opens new session · cwd=~/code/acme-web · agent=claude-code · types "plan a fix for the empty onboarding state"
Agent
Reads source, drafts plan. Realises it needs a current-state screenshot → calls capture_region("current empty state")
NudgeAI
Pops the region overlay with caption. User drags box. Returns PNG path to agent.
Agent
Composes markdown with mermaid diagram + asset ref. Calls render_plan(markdown, assets).
NudgeAI
Templates → styled HTML. Posts to Plan sub-tab. Makes it active. Returns plan_id.
Agent
Calls await_plan_decision(plan_id) — blocks.
User
Reads plan. Either clicks Approve or types in Comment box ("split step 2 into two PRs").
NudgeAI
Resolves the awaiting tool call with the decision + comment.
Agent
If approved → moves to implementing. If commented → revises and loops to render_plan again.
Verify phase
Agent
Finishes a code change. Calls open_preview("localhost:3000/onboarding", select_mode=true).
NudgeAI
Loads URL in Preview sub-tab. Engages SELECT mode. Activates the sub-tab.
Agent
Calls await_element_feedback() — blocks. Optionally posts a banner via post_chat: "take a look — anything off?"
User
Clicks the empty-state container. Box highlights. Floating composer appears with the element's component name tag. Types: "icon should be a folder, not envelope".
NudgeAI
Captures: cropped PNG of the box, CSS selector, outerHTML excerpt, bbox. Bundles with text. Resolves the awaiting call.
Agent
Maps selector → source file. Edits. Calls reload_preview(). Then either await_element_feedback again (more rounds) or finishes the session.
User
Either keeps clicking elements + commenting, or closes the session — at which point NudgeAI flushes the transcript + feedback to disk.
Notes
  • Symmetry: both phases are "agent renders → blocks on user → user gates → resolved with payload". Same pattern, two surfaces.
  • Idle states are fine. The agent doesn't have to enter plan or verify phase. A session can also be just "chat with the agent in the terminal" — render pane stays idle. The new surfaces are opt-in.
  • The terminal isn't "below" plan/verify. It stays live the whole time. User can ⌃C the agent and run shell commands mid-loop.
